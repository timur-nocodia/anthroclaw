# Plan 2 — LCM Core (Lossless Context Management Plugin)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Полный TS-порт hermes-lcm как self-contained Claude Code Plugin (`plugins/lcm/`) на нашем plugin framework из Plan 1. Иммутабельный per-agent SQLite-store + DAG саммаризаций (D0→D1→D2+) + 3-level escalation с гарантированной сходимостью + 6 retrieval-тулзов агенту + интеграция в anthroclaw turn-цикл.

**Architecture:** Плагин = workspace-package в `plugins/lcm/`. Все LLM-вызовы суммаризатора идут через `ctx.runSubagent()` (= `query()` SDK). Регистрация — через `ctx.registerContextEngine({ compress, assemble })`. Mirror — через `ctx.registerHook('on_after_query')`. Тулзы — через `ctx.registerMcpTool()`. Anthroclaw делегирует в плагин через `compressor.ts` (compress) + `gateway.ts` `on_before_query`-точку (assemble). При выключенном плагине — фолбэк на legacy.

**Tech Stack:** TypeScript, `better-sqlite3` (синхронный SQLite + FTS5), zod (config), tiktoken (опционально, char-fallback), `@anthropic-ai/claude-agent-sdk` (только через `ctx.runSubagent`), vitest.

**Reference:** `reference-projects/hermes-lcm/` (Python implementation). Многие файлы — прямой port. План указывает источник для сложных модулей, требует test-first для всего.

**Spec reference:** `docs/superpowers/specs/2026-04-28-lcm-plugin-design.md` §4-§7, §9, §11. Plan 1 (`docs/superpowers/plans/2026-04-28-plugin-framework.md`) — plugin framework, уже мерджнут в main. Plan 3 (UI) — позже.

-- ─────────────────────────────────────────────────────────────────
-- LCM SQLite Schema (managed via versioned migrations in bootstrap.ts)
-- ─────────────────────────────────────────────────────────────────

-- 1) Иммутабельный лог сообщений (append-only)
CREATE TABLE IF NOT EXISTS messages (
  store_id        INTEGER PRIMARY KEY AUTOINCREMENT,
  session_id      TEXT NOT NULL,
  source          TEXT NOT NULL DEFAULT 'unknown',
  role            TEXT NOT NULL,
  content         TEXT NOT NULL,
  tool_call_id    TEXT,
  tool_calls_json TEXT,
  tool_name       TEXT,
  ts              INTEGER NOT NULL,
  token_estimate  INTEGER NOT NULL DEFAULT 0,
  pinned          INTEGER NOT NULL DEFAULT 0
);
CREATE INDEX IF NOT EXISTS idx_messages_session ON messages(session_id, store_id);
CREATE INDEX IF NOT EXISTS idx_messages_source  ON messages(source);

-- FTS5 в режиме external content поверх messages.content
CREATE VIRTUAL TABLE IF NOT EXISTS messages_fts USING fts5(
  content,
  content='messages',
  content_rowid='store_id',
  tokenize='porter unicode61'
);

CREATE TRIGGER IF NOT EXISTS msg_fts_insert AFTER INSERT ON messages BEGIN
  INSERT INTO messages_fts(rowid, content) VALUES (new.store_id, new.content);
END;
CREATE TRIGGER IF NOT EXISTS msg_fts_delete AFTER DELETE ON messages BEGIN
  INSERT INTO messages_fts(messages_fts, rowid, content) VALUES('delete', old.store_id, old.content);
END;
CREATE TRIGGER IF NOT EXISTS msg_fts_update AFTER UPDATE ON messages BEGIN
  INSERT INTO messages_fts(messages_fts, rowid, content) VALUES('delete', old.store_id, old.content);
  INSERT INTO messages_fts(rowid, content) VALUES (new.store_id, new.content);
END;

-- 2) DAG свёрток
CREATE TABLE IF NOT EXISTS summary_nodes (
  node_id              TEXT PRIMARY KEY,
  session_id           TEXT NOT NULL,
  depth                INTEGER NOT NULL,
  summary              TEXT NOT NULL,
  token_count          INTEGER NOT NULL,
  source_token_count   INTEGER NOT NULL,
  source_ids_json      TEXT NOT NULL,
  source_type          TEXT NOT NULL CHECK (source_type IN ('messages', 'nodes')),
  earliest_at          INTEGER NOT NULL,
  latest_at            INTEGER NOT NULL,
  created_at           INTEGER NOT NULL,
  expand_hint          TEXT
);
CREATE INDEX IF NOT EXISTS idx_nodes_session_depth ON summary_nodes(session_id, depth, created_at);

CREATE VIRTUAL TABLE IF NOT EXISTS nodes_fts USING fts5(
  summary,
  content='summary_nodes',
  content_rowid='rowid',
  tokenize='porter unicode61'
);

CREATE TRIGGER IF NOT EXISTS node_fts_insert AFTER INSERT ON summary_nodes BEGIN
  INSERT INTO nodes_fts(rowid, summary) VALUES (new.rowid, new.summary);
END;
CREATE TRIGGER IF NOT EXISTS node_fts_delete AFTER DELETE ON summary_nodes BEGIN
  INSERT INTO nodes_fts(nodes_fts, rowid, summary) VALUES('delete', old.rowid, old.summary);
END;
CREATE TRIGGER IF NOT EXISTS node_fts_update AFTER UPDATE ON summary_nodes BEGIN
  INSERT INTO nodes_fts(nodes_fts, rowid, summary) VALUES('delete', old.rowid, old.summary);
  INSERT INTO nodes_fts(rowid, summary) VALUES (new.rowid, new.summary);
END;

-- 3) Lifecycle state
CREATE TABLE IF NOT EXISTS lcm_lifecycle_state (
  conversation_id              TEXT PRIMARY KEY,
  current_session_id           TEXT,
  last_finalized_session_id    TEXT,
  current_frontier_store_id    INTEGER,
  last_finalized_frontier_id   INTEGER,
  debt_kind                    TEXT,
  debt_size_estimate           INTEGER,
  updated_at                   INTEGER,
  reset_at                     INTEGER,
  finalized_at                 INTEGER
);

-- 4) Метаданные (schema version, etc.)
CREATE TABLE IF NOT EXISTS schema_meta (
  key   TEXT PRIMARY KEY,
  value TEXT NOT NULL
);

- [ ] **Step 4: Implement bootstrap.ts**

`plugins/lcm/src/db/bootstrap.ts`:

```typescript
import type { Database } from 'better-sqlite3';
import { readFileSync } from 'node:fs';
import { fileURLToPath } from 'node:url';
import { dirname, join } from 'node:path';

export const SCHEMA_VERSION = 1;

const __dirname = dirname(fileURLToPath(import.meta.url));

export function bootstrap(db: Database): void {
  // Read schema.sql
  // In compiled dist/, schema.sql is alongside JS — need to copy at build OR embed.
  // For simplicity at v0.1.0, embed via inline string. Build step copies schema.sql to dist/.
  // (Alternative: import.meta + readFileSync in dev, fallback in prod. We use copy-asset approach.)
  const schemaPath = join(__dirname, 'schema.sql');
  const schemaSql = readFileSync(schemaPath, 'utf-8');

  db.pragma('journal_mode = WAL');
  db.pragma('foreign_keys = ON');

  db.exec(schemaSql);

  // Schema version handling
  const currentVersion = getSchemaVersion(db);
  if (currentVersion === 0) {
    // Fresh DB — set to current
    setSchemaVersion(db, SCHEMA_VERSION);
  } else if (currentVersion < SCHEMA_VERSION) {
    runMigrations(db, currentVersion, SCHEMA_VERSION);
    setSchemaVersion(db, SCHEMA_VERSION);
  } else if (currentVersion > SCHEMA_VERSION) {
    throw new Error(
      `LCM database has newer schema version (${currentVersion}) than this plugin supports (${SCHEMA_VERSION}). ` +
      `Forward-compat upgrades are not supported — update the plugin.`
    );
  }
}

export function getSchemaVersion(db: Database): number {
  // schema_meta is created by schema.sql, so this always works after bootstrap
  try {
    const row = db.prepare(`SELECT value FROM schema_meta WHERE key = 'schema_version'`).get() as { value: string } | undefined;
    return row ? parseInt(row.value, 10) : 0;
  } catch {
    return 0;
  }
}

function setSchemaVersion(db: Database, v: number): void {
  db.prepare(`INSERT OR REPLACE INTO schema_meta(key, value) VALUES('schema_version', ?)`).run(String(v));
}

function runMigrations(db: Database, from: number, to: number): void {
  for (let v = from + 1; v <= to; v++) {
    const migration = MIGRATIONS[v];
    if (!migration) {
      throw new Error(`No migration registered for schema version ${v}`);
    }
    migration(db);
  }
}

const MIGRATIONS: Record<number, (db: Database) => void> = {
  // Future migrations: 2: (db) => { db.exec('ALTER TABLE ...') }
};
```

/**
 * Token estimation. Uses tiktoken (cl100k_base) when available; otherwise
 * falls back to char/4 heuristic (matches hermes-lcm behavior).
 *
 * tiktoken is NOT a hard dep — keeps install footprint small. Plugin
 * works without it (slightly less accurate token counts, no functional impact).
 */

interface Tokenizer {
  encode(text: string): number[];
}

let cached: Tokenizer | null | undefined;

function loadTokenizer(): Tokenizer | null {
  if (cached !== undefined) return cached;
  try {
    // Use dynamic import via require-style for optional dep
    // eslint-disable-next-line @typescript-eslint/no-require-imports
    const tiktoken = require('tiktoken');
    cached = tiktoken.encoding_for_model('gpt-4') ?? tiktoken.get_encoding('cl100k_base');
    return cached;
  } catch {
    cached = null;
    return null;
  }
}

export function hasNativeTokenizer(): boolean {
  return loadTokenizer() !== null;
}

export interface EstimateOpts {
  forceFallback?: boolean;
}

export function estimateTokens(text: string, opts: EstimateOpts = {}): number {
  if (text.length === 0) return 0;
  if (!opts.forceFallback) {
    const tk = loadTokenizer();
    if (tk) {
      try { return tk.encode(text).length; } catch { /* fall through */ }
    }
  }
  return Math.ceil(text.length / 4);
}
```

  token_estimate: number;
  pinned: boolean;
}

export interface SearchOpts {
  source?: string;
  sort?: 'relevance' | 'recency' | 'hybrid';
  limit?: number;
  /** Only consider messages with store_id > minStoreId. */
  minStoreId?: number;
  /** Restrict to a session_id (or null = all sessions). */
  sessionId?: string | null;
}

export interface SearchResult {
  store_id: number;
  session_id: string;
  source: string;
  role: string;
  ts: number;
  snippet: string;     // FTS5 snippet() output
  rank: number;        // BM25 rank from FTS5
}

export class MessageStore {
  constructor(db: Database);

  /** Append a message. Returns new store_id. */
  append(msg: InboundMessage): number;

  /** Fetch a single message by store_id. */
  get(storeId: number): StoredMessage | null;

  /** Fetch many by ids. Order matches input. Missing ids skipped. */
  getMany(storeIds: number[]): StoredMessage[];

  /** Fetch all messages in session, ordered by store_id. */
  listSession(sessionId: string): StoredMessage[];

  /** FTS5 search. Falls back to LIKE for CJK/emoji/unbalanced quotes. */
  search(query: string, opts?: SearchOpts): SearchResult[];

  /** Last store_id (frontier). */
  maxStoreId(): number;

  /** Count of messages in session. */
  countInSession(sessionId: string): number;

  /** Replace tool-result content with placeholder (only allowed mutation — opt-in transcript GC). */
  gcExternalizedToolResult(storeId: number, placeholder: string): void;

  /** Pin/unpin message. */
  setPinned(storeId: number, pinned: boolean): void;
}

import { join } from 'node:path';
import { mkdirSync } from 'node:fs';
import { LCMConfigSchema, resolveConfig, type ResolvedLCMConfig } from './config.js';
import { bootstrap } from './db/bootstrap.js';
import { MessageStore } from './store.js';
import { SummaryDAG } from './dag.js';
import { LifecycleManager } from './lifecycle.js';
import { LCMEngine } from './engine.js';
import { createMirrorHook } from './hooks/mirror.js';
import { createGrepTool } from './tools/grep.js';
import { createDescribeTool } from './tools/describe.js';
import { createExpandTool } from './tools/expand.js';
import { createExpandQueryTool } from './tools/expand-query.js';
import { createStatusTool } from './tools/status.js';
import { createDoctorTool } from './tools/doctor.js';

interface PerAgentState {
  db: Database.Database;
  store: MessageStore;
  dag: SummaryDAG;
  lifecycle: LifecycleManager;
  engine: LCMEngine;
  config: ResolvedLCMConfig;
}

export async function register(ctx: PluginContext): Promise<PluginInstance> {
  ctx.logger.info({ version: ctx.pluginVersion }, 'LCM plugin loading');

  const dbDir = join(ctx.dataDir, 'lcm-db');
  mkdirSync(dbDir, { recursive: true });

  const perAgent = new Map<string, PerAgentState>();

  function getOrCreateForAgent(agentId: string): PerAgentState {
    let state = perAgent.get(agentId);
    if (!state) {
      const agentConfig = ctx.getAgentConfig(agentId) as { plugins?: { lcm?: unknown } } | undefined;
      const globalConfig = ctx.getGlobalConfig() as { plugins?: { lcm?: { defaults?: unknown } } } | undefined;
      const config = resolveConfig(
        globalConfig?.plugins?.lcm?.defaults ?? {},
        agentConfig?.plugins?.lcm ?? {},
      );

      const db = new Database(join(dbDir, `${agentId}.sqlite`));
      bootstrap(db);
      const store = new MessageStore(db);
      const dag = new SummaryDAG(db);
      const lifecycle = new LifecycleManager(db);
      const engine = new LCMEngine({
        store, dag, lifecycle,
        runSubagent: ctx.runSubagent.bind(ctx),
        config,
        logger: ctx.logger,
      });
      state = { db, store, dag, lifecycle, engine, config };
      perAgent.set(agentId, state);
    }
    return state;
  }

  // ContextEngine — implements compress/assemble
  const engineFacade: ContextEngine = {
    async compress(input) {
      const state = getOrCreateForAgent(input.agentId);
      if (!state.config.enabled) return null;
      try {
        const out = await state.engine.compress(input as never);
        return { messages: out.messages };
      } catch (err) {
        ctx.logger.warn({ err, agentId: input.agentId }, 'lcm compress failed, falling back');
        return null;
      }
    },
    async assemble(input) {
      const state = getOrCreateForAgent(input.agentId);
      if (!state.config.enabled) return null;
      try {
        const out = await state.engine.assemble(input as never);
        return { messages: out.messages };
      } catch (err) {
        ctx.logger.warn({ err, agentId: input.agentId }, 'lcm assemble failed, pass-through');
        return null;
      }
    },
  };
  ctx.registerContextEngine(engineFacade);

  // Mirror hook
  const mirrorDeps = {
    engine: { ingest: (sessionId: string, source: string, msgs: unknown[]) => {
      // dispatch to per-agent engine via payload.agentId — implemented inside hook
    } } as unknown as LCMEngine,
    config: null as unknown as ResolvedLCMConfig,
    logger: ctx.logger,
  };
  ctx.registerHook('on_after_query', (payload) => {
    const agentId = payload.agentId as string | undefined;
    if (!agentId) return;
    const state = getOrCreateForAgent(agentId);
    if (!state.config.enabled) return;
    createMirrorHook({ engine: state.engine, config: state.config, logger: ctx.logger })(payload);
  });

  // MCP tools (read agent config at registration time using a placeholder agentId resolver — actual
  // session resolution happens inside tool handlers via ctx if needed)
  // For Plan 2 v0.1.0: tools are registered per-plugin (not per-agent). They internally resolve
  // current agent from runtime hints. Implementer: revisit this if needed.

  // Helper to wrap tool registration with config gating
  const registerIfEnabled = (toolName: keyof ResolvedLCMConfig['tools'], toolFactory: () => unknown) => {
    // We don't have an agent-id at registration time. Tools are registered globally.
    // Per-agent tool gating happens via the registry's enabledByAgent + per-agent config check
    // inside each tool handler. v0.1.0 simplification: register all tools, check config inside handler.
    ctx.registerMcpTool(toolFactory() as never);
  };

  // Build tools (each tool factory takes agentId-resolver — implementer wires this through)
  // For now, all tools are registered; per-tool config gating happens inside handler.
  registerIfEnabled('grep', () => createGrepTool({ getOrCreateForAgent }));
  registerIfEnabled('describe', () => createDescribeTool({ getOrCreateForAgent }));
  registerIfEnabled('expand', () => createExpandTool({ getOrCreateForAgent }));
  registerIfEnabled('expand_query', () => createExpandQueryTool({ getOrCreateForAgent, runSubagent: ctx.runSubagent.bind(ctx) }));
  registerIfEnabled('status', () => createStatusTool({ getOrCreateForAgent }));

  return {

        try { state.db.close(); } catch (err) { ctx.logger.warn({ err }, 'lcm db close error'); }

## Task 22: skills/lcm-usage.md — agent-facing instruction

**Files:**

### Content

```markdown
---
name: lcm-usage
description: When and how to use Lossless Context Management tools (lcm_*).
---

# Using LCM Tools

You have access to a session-scoped DAG of compressed history. Use these tools when the user references something from earlier in the session that was compacted out of your active context.

## When to use which tool

- **`lcm_grep`** — search by keyword. Use when user asks "что мы говорили про X" / "find where we discussed Y". Returns top matches across raw messages and compressed summaries.

- **`lcm_describe`** — preview metadata of a node without loading content. Use to scope before drilling. With no args returns session overview.

- **`lcm_expand`** — fetch source content for a node. After `lcm_grep` returns a node_id, call `lcm_expand` to drill down. For deep nodes (D2/D3), expand iteratively (D2 → child D1s → child D0s → raw messages).

- **`lcm_expand_query`** — RAG-style: provide a natural-language `prompt` and the tool finds relevant nodes, expands them, and answers. Use for "summarize what we decided about Y across the whole session". Faster than manual grep + expand.

- **`lcm_status`** — diagnostic: how big is the DAG, how many compressions ran, last compaction time. Use rarely — for self-diagnosis or when user explicitly asks.

- **`lcm_doctor`** — health check (orphans, FTS sync, integrity). Use only when something seems wrong or operator asks.

## Distinction from `memory_*` tools

- `memory_*` tools = **long-term, cross-session memory** (wiki entries, persistent facts).
- `lcm_*` tools = **current session DAG** (compressed history of THIS conversation).

## LCM Plugin (Lossless Context Management)

Optional plugin (`plugins/lcm/`) implementing hierarchical context compression with full byte-exact recovery of source messages. Based on [hermes-lcm](https://github.com/stephenschoettler/hermes-lcm).

### How it works

When session token count crosses `compress_threshold_tokens`, the plugin:
1. Mirrors all messages into per-agent SQLite (immutable append-only log).
2. Groups older messages into chunks, summarizes each via L1→L2→L3 escalation (L3 is deterministic — never calls LLM).
3. Builds D0 nodes (raw chunk summaries), D1 (4 D0s → 1 D1), D2 (4 D1s → 1 D2), etc — a hierarchical DAG.
4. Re-assembles prompt as `[system, ...top-N anchors per depth, ...fresh tail]` capped at `assembly_cap_tokens`.

### 6 retrieval tools

The agent gets these tools when LCM is enabled:

- `lcm_grep` — FTS over messages + summaries. Use for "what did we discuss about X earlier".
- `lcm_describe` — preview node metadata without loading.
- `lcm_expand` — drill into a node (D2 → D1 children → D0 → raw messages).
- `lcm_expand_query` — RAG-style: prompt + context window assembled from matching nodes.
- `lcm_status` — diagnostic.
- `lcm_doctor` — health check + double-gated cleanup.

### Configuration (full reference)

Defaults in `config.yml`, per-agent override in `agent.yml`:

```yaml
# config.yml — global defaults
plugins:
  lcm:
    defaults:
      enabled: false              # opt-in
      triggers:
        compress_threshold_tokens: 40000
        fresh_tail_count: 64
        assembly_cap_tokens: 160000
      summarizer:
        summary_timeout_ms: 60000
      escalation:
        l1_budget_pct: 0.20
        l3_truncate_tokens: 512
      dag:
        condensation_fanin: 4
      lifecycle:
        carry_over_retain_depth: 2

# agent.yml — per-agent enable
plugins:
  lcm:
    enabled: true                 # required to activate
```

### When NOT to use LCM

- Short-lived agents (sessions never grow past threshold).
- Agents where context window is naturally small (Haiku-only, etc).
- Memory-constrained gateways (one SQLite per agent grows ~1KB per message).

### Storage

Per-agent SQLite at `data/lcm-db/{agentId}.sqlite`. Schema versioned via `bootstrap.ts`. Backups created on `lcm_doctor apply: true` (under double gate) at `data/lcm-db/backups/`.

### Lossless invariant

The defining property: from any D2/D3/etc node, the recursive `source_ids` chain leads back to original byte-exact messages in the immutable store. Verified by the `@lossless` test suite (4 scenarios: drill-down, source-lineage filter, carry-over preservation, restart-survives).
```
